Template data

The Webpack Handlebars plugin that we use comes with a very handy utility that allows us to pass in JSON files as template data.

What is JSON? Read more here: https://www.w3schools.com/whatis/whatis_json.asp

Please navigate to: src/data. Here is where our template data JSON files reside. You can edit, remove or add your own to this folder. We have organised the JSON files in subfolders, but please note that the subfolders are not required and are not referenced by Handlebars when you output the data. Let's look at what that means. Please note that each time you make a change to your JSON files, you have to restart your Webpack devserver. This is because the data JSON files are passed in when Webpack starts. It does not monitor the files for any changes.

Controlling layout partials

Another huge benefit of using our JSON data setup is that we can easily control core layout options for our HTML pages. So instead of having to change the header partial for every single page, most pages can have their header or footer changed from a single line inside the config.json file. This is what controls which header partial is output: "navbar": "navbar-one/navbar-light". So if you wanted to change to navbar-two, you would amend the code to be: "navbar": "navbar-two/navbar-light". Then quick Webpack, restart it, and now your template will display navbar-two as the navbar.

We do this for two reasons: one, so that some of our key pages can showcase our different headers and footers. And two, so that both methods of outputting the header/footer partials are clear to you. This means you can control the header and footer partial from our config.json for all pages but you have the ability to set custom header or footers for specific pages as well.

The reason we have setup the template in this manner is because Handlebars doesn't offer layout functionality that other templating engines such as Nunjucks does. We wanted it to be as easy as possible for our customers to make core layout changes to all pages without the need to open up every single HTML page.
