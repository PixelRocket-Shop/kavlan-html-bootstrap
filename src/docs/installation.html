Installation

You only need to do the following steps if you intend to run Webpack and recompile the template. Please also ensure that you have Node.js installed (unsure? read the "Requirements" section in the "Getting Started" section of these helpdocs).

Step One: open up the template folder in your command line. Ensure you are inside the root of the template.

Step Two: in your command line, run npm install. This will install all the dependancies listed in your package.json file.

Step Three: in your command line, run npm start. That's it. Your browser will now open and the "dev" version of the site will be loaded.

Please note that when you run npm start, Webpack devserver is started. This means any changes you make will not visible in the template's dist folder until you compile it. Webpack devserver holds the files in memory. So make all the changes to the files inside the src folder, view them in your browser, and then when and if you require access to the compiled code, follow the steps below to compile your template.

Build template

As mentioned above, when you run Webpack devserver, the files are kept in memory. You might at some stage want to get access to the compiled version of your updated code. To do that, follow these steps:

Step One: if you currently have the site running through Webpack devserver, kill that process by pressing the CTRL + C buttons (make sure you are in your command line).

Step Two: in your command line, type npm run build.

And that's it. The existing dist folder will now be deleted by Webpack, and the updated code will be compiled and output to a new dist folder.
